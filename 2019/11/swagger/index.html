Swagger

2019年十一月7日 发布在 swagger

Swagger是用于前后端分离的api框架,基于RestFul Api风格的文档在线自动生成工具.
可以实现Api文档与Api定义同步更新,支持在线测试.
注意:在正式发布的时候关闭swagger!!

		<dependency>

基本配置

@Configuration
@EnableSwagger2
public class Swagger2 {
    @Bean
    public Docket createRestApi() {
        ParameterBuilder tokenPar = new ParameterBuilder();
        List<Parameter> pars = new ArrayList<>();
        tokenPar.name("token").description("令牌").modelRef(new ModelRef("string")).parameterType("header").required(false).build();
        pars.add(tokenPar.build());
        return new Docket(DocumentationType.SWAGGER_2)
                //文档信息
                .apiInfo(apiInfo())
                //设置组名
                .groupName("xxx")
                //是否启动swagger默认ture,如果为false则不能再浏览器中访问swagger
                .enable(ture)  
                .select()
                //RequestHandlerSelectors配置扫描接口方式
                   //basePackage指定要扫描的包
                   //any全部扫描  //none不扫描
                   //withClassAnnotation扫描类上的注解,参数是反射对象xxx注解.calss
                   //withMethodAnnotation扫描方法上的注解,参数是反射对象xxx注解.calss
                .apis(RequestHandlerSelectors.basePackage("com.ruowei.groupbuying"))
                //过滤(得到滤下的)   
                    //.ant("/xxx/xx") 路径
                .paths(PathSelectors.any())
                .build()
                //给请求添加全局公共参数(如token)
                .globalOperationParameters(pars);
    }
    //构建 api文档的详细信息函数,注意这里的注解引用的是哪个
    private ApiInfo apiInfo() {
        return new ApiInfoBuilder()
                //页面标题
                .title("Spring Boot 测试使用 Swagger2 构建RESTful API")
                //作者信息
                .contact(new Contact("name", "url", "email"))
                //版本号
                .version("1.0")
                //描述
                .description("API 描述")
                .build();
    }
}

配置全局参数(如token)

非全局、无需重复输入的Head参数（Token）配置

@Configuration
@EnableSwagger2
public class Swagger {
    @Bean
    public Docket api() {
        return new Docket(DocumentationType.SWAGGER_2).
                useDefaultResponseMessages(false)
                .select()
                .apis(RequestHandlerSelectors.any())
                .paths(PathSelectors.regex("^(?!auth).*$"))
                .build()
                .securitySchemes(securitySchemes())
                .securityContexts(securityContexts())
                ;
    }
//通过Swagger2的securitySchemes配置全局参数：如下列代码所示，securitySchemes的
    //ApiKey中增加一个名为“Authorization”，type为“header”的参数。
    private List<ApiKey> securitySchemes() {
        return newArrayList(
                new ApiKey("Authorization", "Authorization", "header"));
    }
//在Swagger2的securityContexts中通过正则表达式，设置需要使用参数的接口（或者说，是去除掉不需要使用参数的接口），
//如下列代码所示，通过PathSelectors.regex("^(?!auth).*$")，
//所有包含"auth"的接口不需要使用securitySchemes。
//即不需要使用上文中设置的名为“Authorization”，type为“header”的参数。
    private List<SecurityContext> securityContexts() {
        return newArrayList(
                SecurityContext.builder()
                        .securityReferences(defaultAuth())
                        .forPaths(PathSelectors.regex("^(?!auth).*$"))
                        .build()
        );
    }

    List<SecurityReference> defaultAuth() {
        AuthorizationScope authorizationScope = new AuthorizationScope("global", "accessEverything");
        AuthorizationScope[] authorizationScopes = new AuthorizationScope[1];
        authorizationScopes[0] = authorizationScope;
        return newArrayList(
                new SecurityReference("Authorization", authorizationScopes));
    }
}

通过配置文件改变来设置swagger是否启动

希望swagger只在开发环境启动,而生产环境不启动 2. 使用注解@Profile({“dev”,“test”})表示在dev或test配置环境开启.在默认配置文件中使用spring.profiles.active=dev来切换不同的配置环境.
3. 设置公共字段,通过手动改变字段的值来开启关闭swagger.
