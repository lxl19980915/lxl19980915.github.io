通过简单的git命令,将本地代码上传到远程仓库中.

一.注册github或coding


二.下载git

git官方下载地址 傻瓜式安装过程,一直下一步就行.

三.配置git

  1. 在任意位置右键,启动Git Bash客户端

  2. 设置公匙

    $ ssh-keygen -t rsa -C "youremail@example.com"

    youremail@example.com就是你在coding.net或github上注册的邮箱，之后需要确认Key的保存路径和密码，一般我们直接敲回车，默认就可以了。这样子在用户根目录下回生成 .ssh 文件夹.生成公钥以后，在.ssh文件夹中会有id_rsa和id_rsa.pub文件.
    公钥就在id_rsa.pub文件中.

  3. 将本机ssh公匙添加到github或coding

  4. 配置本地用户名和邮箱

    $ git config --global user.name “your_username” //设置用户名
$ git config --global user.email “your_Email_address”  //设置邮箱地址(建议用注册coding.net的邮箱)

四.将本地项目推到远程仓库

  1. 创建本地仓库 在本地项目的根目录,单击右键—〉 git bash here

    git init

  2. 添加、提交文件到本地仓库

    git add . 或者 git add filename
git commit –m "项目描述"

  3. 上传到远程仓库

    git remote add origin 项目的ssh公匙
git push origin master

五.git的常用命令

git status   //当前状态  当前分支,已提交信息,需要提交文件 (提示信息)
git init     //初始化 会生成一个.git文件
git add .    //追踪该目录下所有文件(就是放入暂存区),可把.换成具体文件如a.txt
git commit –m "项目描述" //提交文件到本地仓库 -m "xx"附带注释
git remote add origin 项目的ssh公匙  //将本地仓库与远程仓库连接
git push origin master      //将本地仓库代码提交到远程仓库的master分支
git clone 项目的ssh公匙  //克隆远程仓库项目到本地
git push        //推,上传
git pull        //拉,下载
